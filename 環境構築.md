# 環境構築手順 (Setup Guide)

Sullivanアプリケーションの開発環境構築手順です。

## 前提条件
- Node.js (v18以上推奨)
- PostgreSQL (ローカル or Supabase)
- Supabase project (Auth用)
- Google Cloud Account (Gemini API, Drive API用)
- Upstash (QStash, Redis)
- Python 3 (QRコード読取用)

## 1. 依存関係のインストール

```bash
npm install
# Python dependencies (for QR code reading)
pip3 install opencv-python
```

## 2. データベースの準備

### ローカルPostgreSQLを使う場合

```bash
# PostgreSQLサービスの起動 (例: Homebrew使用時)
brew services start postgresql@18

# データベースの作成
createdb sullivan_db
```

### Supabaseを使う場合
Supabaseの「Connect」から接続文字列を取得し、以下のように使い分けます。
- **Transaction Mode (Port 6543)**: `DATABASE_URL` に使用（末尾に `?pgbouncer=true`）
- **Session Mode (Port 5432)**: `DIRECT_URL` に使用

## 3. 環境変数の設定

プロジェクトルートに `.env` ファイルを作成し、以下の変数を設定してください。

```env
# Database
DATABASE_URL="postgresql://[USER]@localhost:5432/sullivan_db"
DIRECT_URL="postgresql://[USER]@localhost:5432/sullivan_db"

# Supabase Auth
NEXT_PUBLIC_SUPABASE_URL="https://your-project.supabase.co"
NEXT_PUBLIC_SUPABASE_ANON_KEY="your_supabase_anon_key"
SUPABASE_SERVICE_ROLE_KEY="your_supabase_service_role_key"

# AI & Grading (Google Gemini API)
GEMINI_API_KEY="your_gemini_api_key_here"
# Optional: override Gemini model
GEMINI_MODEL="gemini-2.5-pro"

# Google Drive Integration (for Scan Grading)
# 答案スキャンフォルダのID
DRIVE_FOLDER_ID="your_google_drive_folder_id"
# Optional: webhook channel validation
DRIVE_WEBHOOK_CHANNEL_ID="your_drive_webhook_channel_id"

# Google Application Credentials (production / Cloud Run)
# ローカル開発では service-account.json をプロジェクトルートに置く方法でも可
GOOGLE_APPLICATION_CREDENTIALS="/secrets/google-creds"

# Job Queue (Upstash QStash)
QSTASH_TOKEN="your_qstash_token"
# Optional but recommended: signature verification keys
QSTASH_CURRENT_SIGNING_KEY="your_qstash_current_signing_key"
QSTASH_NEXT_SIGNING_KEY="your_qstash_next_signing_key"
APP_URL="https://your-app-url.com" # (or ngrok url for local dev)

# Drive Webhook State (Upstash Redis)
UPSTASH_REDIS_REST_URL="https://your-upstash-redis-url"
UPSTASH_REDIS_REST_TOKEN="your_upstash_redis_token"

# Internal API (optional)
INTERNAL_API_SECRET="your_internal_api_secret"
```

**Google Cloud Service Account**:
Google Drive APIへのアクセス権限を持つ `service-account.json` をプロジェクトルートに配置してください。
本番環境では `GOOGLE_APPLICATION_CREDENTIALS` でシークレットのマウント先を指定します。

## 4. マイグレーションとシードデータの投入

データベーススキーマの適用と、初期データの投入を行います。

```bash
npx prisma migrate dev
npx prisma db seed
```

※ Supabase Authにユーザーを作成するため、`SUPABASE_SERVICE_ROLE_KEY` の設定が必要です。
未設定の場合はDBのみ作成され、ログイン用ユーザーは作成されません。

### 初期アカウント
- **管理者**: `A0001` / `password123`
- **講師**: `T0001` / `password123`
- **生徒**: `S0001` / `password123`

初回ログイン時はパスワード変更が必須です。

## 5. 開発サーバーの起動

```bash
npm run dev
```

サーバーが起動したら、 [http://localhost:3000](http://localhost:3000) にアクセスしてください。

## 6. Drive Webhook セットアップ (任意)

Driveのリアルタイム検知を使う場合は、`APP_URL` を外部公開できるURLに設定した上で、
以下を一度実行してWatchを登録してください。

```bash
curl -X POST https://your-app-url.com/api/drive/watch/setup
```

Watchは約24時間で失効するため、Cloud Scheduler等で
`POST /api/drive/watch/renew` を12時間おきに叩く運用が推奨です。
