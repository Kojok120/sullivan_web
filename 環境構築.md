
# 環境構築手順

Sullivan のローカル開発環境をセットアップする手順です。

## 1. 前提

- Node.js 20 以上（推奨）
- npm
- PostgreSQL（ローカル or Supabase）
- Supabase プロジェクト（Auth 利用）
- Google Cloud（Gemini / Drive を使う場合）
- Python 3（QR 解析をローカルで行う場合）

## 2. 依存関係のインストール

```bash
npm install
```

QR ローカル解析（OpenCV）を使う場合は、`/usr/bin/python3` で `cv2` が import できる状態にしてください。

```bash
/usr/bin/python3 -c "import cv2; print(cv2.__version__)"
```

## 3. 環境変数設定

`.env.local` を作成し、まずは以下を設定します。

```env
# Database
DATABASE_URL="postgresql://..."
DIRECT_URL="postgresql://..."

# Supabase
NEXT_PUBLIC_SUPABASE_URL="https://...supabase.co"
NEXT_PUBLIC_SUPABASE_ANON_KEY="..."
SUPABASE_SERVICE_ROLE_KEY="..."

# Gemini
GEMINI_API_KEY="..."
GEMINI_MODEL="gemini-3.1-pro-preview"
GEMINI_CHAT_MODEL="gemini-3.1-pro-preview"
GEMINI_CHAT_FALLBACK_MODEL="gemini-3.1-pro-preview"
```

Drive採点やWorker連携まで行う場合は追加で設定します。

```env
# Drive / Webhook
DRIVE_FOLDER_ID="..."
APP_URL="http://localhost:3000"
INTERNAL_API_SECRET="..."
DRIVE_WEBHOOK_TOKEN="..."
DRIVE_WEBHOOK_CHANNEL_ID="..." # 任意

# Cloud Tasks / Worker
GOOGLE_CLOUD_PROJECT_ID="..."
CLOUD_TASKS_LOCATION="asia-northeast1"
GRADING_TASK_QUEUE="sullivan-grading"
DRIVE_CHECK_TASK_QUEUE="sullivan-drive-check"
CLOUD_TASKS_CALLER_SERVICE_ACCOUNT="sullivan-runtime@<PROJECT_ID>.iam.gserviceaccount.com"
GRADING_WORKER_URL="http://localhost:8080"
DRIVE_WATCH_STATE_KEY="sullivan:drive:watch:state"

# Gemini Live（音声AIチューター）
GEMINI_LIVE_SESSION_SECRET="..."
```

## 4. DBセットアップ

```bash
npx prisma generate
npx prisma migrate dev
npx prisma db seed
```

## 5. 開発サーバー起動

Web サービス:

```bash
npm run dev
```

Worker サービス（別ターミナル）:

```bash
npm run dev:worker
```

- Web: `http://localhost:3000`
- Worker: `http://localhost:8080`（`PORT` 未指定時）

## 6. 動作確認コマンド

```bash
npm run lint
npm run type-check
npm run test
```

## 7. Drive Watch（任意）

Webhook運用を試す場合、`INTERNAL_API_SECRET` を使って Watch を設定します。

```bash
curl -X POST "${APP_URL}/api/drive/watch/setup" \
  -H "Authorization: Bearer ${INTERNAL_API_SECRET}"
```

更新（7日失効対策）:

```bash
curl -X POST "${APP_URL}/api/drive/watch/renew" \
  -H "Authorization: Bearer ${INTERNAL_API_SECRET}"
```

## 8. デプロイ補足

リリースは GitHub Actions による自動デプロイで運用しています（`.sh` 直叩きは非推奨）。

- `dev` ブランチ push → `.github/workflows/deploy-dev.yml`
- `main` ブランチ push → `.github/workflows/deploy-production.yml`
- main へのマージ元は `dev` に制限（`restrict-main-merge-source.yml`）

CI から呼ばれる内部スクリプト（参考、手元から実行することは想定しない）:

- `./deploy-web-DEV.sh` / `./deploy-web-PRODUCTION.sh`
- `./deploy-grading-worker-DEV.sh` / `./deploy-grading-worker-PRODUCTION.sh`

詳細は [docs/deploy_runbook.md](./docs/deploy_runbook.md) を参照してください。
