# Sullivan 機能一覧とロジック詳細

本ドキュメントでは、学習支援システム「Sullivan」の現行実装に基づく機能一覧と、主要ロジックについて記述します。

## 1. システム概要
Sullivanは、プリント学習を主体とした学習支援Webアプリケーションです。
生徒の学習状況（忘却度、苦手単元、進捗）に基づき、最適な問題を自動選出してプリントを作成します。
解答済みのプリントはGoogle Driveにアップロードされ、Webhook + QStash経由で自動採点・分析が行われます。

## 2. ユーザーロール
システムは以下のロールを持ちます。

* **ADMIN (管理者)**: ユーザー/教室/カリキュラム/問題の管理。
* **TEACHER (講師)**: 生徒管理、指導記録、プリント作成。
* **STUDENT (生徒)**: 学習状況の確認、履歴閲覧。
* **PARENT (保護者)**: スキーマ上は存在しますが、現状のUI/権限は未提供です。

## 3. 機能一覧

### 3.1 認証・ユーザー管理
* **ログイン/ログアウト**:
    * Supabase Authを使用。
    * ログインIDを `loginId@sullivan-internal.local` に変換して認証。
    * 役割情報は `app_metadata` を参照し、`middleware.ts` でルート保護。
* **初回パスワード変更**:
    * 既定パスワード利用時は `/force-password-change` へ強制遷移。
* **ユーザー作成/更新**:
    * 管理者がユーザー作成・パスワードリセット可能。
    * 講師は担当教室内の生徒プロフィールを編集可能。

### 3.2 管理者向け機能
* **ユーザー管理**: 作成、検索、権限付与、パスワードリセット。
* **教室管理**: 教室の追加・削除、所属管理。
* **カリキュラム管理**: Subject / CoreProblem / Problem の編集、一括登録。
* **問題管理**: 単独編集、一括登録。

### 3.3 講師向け機能
* **生徒一覧・検索**: 氏名、ID、グループで検索。
* **学習進捗確認**: 生徒別の学習数・正答率・連続学習日数。
* **指導記録 (Guidance)**: 面談/指導の記録・閲覧。
* **プリント作成**:
    * 生徒と教科を選択し、問題セットを自動選出（デフォルト30問）。
    * 解答用紙を含む印刷レイアウトを生成。
    * QRコードに生徒IDと問題IDを埋め込み。

### 3.4 生徒向け機能
* **学習ダッシュボード**:
    * 総学習数、正答率、連続学習日数。
    * 直近30日の学習アクティビティ。
    * 教科別進捗（アンロック済みCoreProblem数）。
* **学習履歴**: セッション単位の履歴一覧と詳細。
* **未視聴解説の通知**: 未視聴の解説動画がある場合に警告表示。
* **採点通知**: SSEで採点完了イベントを受信し、通知表示。

## 4. コアロジック詳細 (The "Sullivan Logic")

### 4.1 プリント生成・問題選出ロジック
実装: `src/lib/print-algo.ts` / `src/lib/progression.ts`

**選出の流れ:**
1. **アンロック判定**: `UserCoreProblemState.isUnlocked` を基準に解放済みCoreProblemを判定。最初のCoreProblemは常にアンロック。
2. **候補抽出**: 「紐づくすべてのCoreProblemがアンロック済み」の問題のみを候補とする。
3. **スコアリング**:
    * **CoreProblem優先度**: `UserCoreProblemState.priority` の合計値。
    * **忘却度**: `UserProblemState.lastAnsweredAt` からの経過日数。
    * **未回答ボーナス**: 未回答問題に加点。
4. **選抜**: スコア上位から指定数（デフォルト30問）を採用。

### 4.2 AI自動採点・分析ロジック
実装: `src/lib/grading-service.ts`

**処理フロー:**
1. **Drive検知**: `/api/grading/webhook` がDriveのPush通知を受信。
2. **ジョブ発行**: `checkDriveForNewFiles` が新規ファイルを抽出し、QStashへキューイング（未設定時は同期処理）。
3. **ファイル処理**: `/api/queue/grading` がファイルを `/tmp` にダウンロード。
4. **QR解析**:
    * 画像ファイルはPython + OpenCVでQR解析。
    * 失敗時やPDFはGeminiでQR解析をフォールバック。
5. **採点**:
    * Geminiが解答を読み取り、A〜D評価と日本語フィードバックを生成。
    * 不正解時は原因となるCoreProblem IDを推定。
6. **保存/更新**:
    * `LearningHistory` に結果保存。
    * `UserProblemState` を更新（A:-10, B:-5, C:+5, D:+10）。
    * `UserCoreProblemState` を更新（正解: 全CoreProblemに-5、不正解: 推定CoreProblemに+5）。
7. **アンロック判定**: 関連CoreProblemに対して `checkProgressAndUnlock` を実行。
8. **通知**: SSEイベント `GRADING_COMPLETED` を発火。

### 4.3 アンロックシステム (Unlock Logic)
* **判定条件**:
    * **解答率**: 回答済み問題数 / 対象問題数 >= 50%
    * **正答率**: 正解数 / 回答済み問題数 >= 60%
* **対象問題**: 「紐づくすべてのCoreProblemがアンロック済み」の問題のみを集計対象とする。
* **解放処理**: 条件を満たしたCoreProblemの次のCoreProblemを自動アンロック。

## 5. データモデル構造 (重要エンティティ)

* **Subject / CoreProblem / Problem**: 教材の階層構造。
* **UserProblemState**: 「生徒×問題」ごとの状態（優先度、最終解答日時、正解フラグ）。
* **UserCoreProblemState**: 「生徒×単元」ごとの状態（優先度、アンロック状況）。
* **LearningHistory**: 解答ログ（評価、フィードバック、グループID）。
* **GuidanceRecord / Classroom**: 指導記録と教室情報。
