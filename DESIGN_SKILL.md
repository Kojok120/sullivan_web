# Product UI Design Skill

プロダクトのデザインガイドライン（DESIGN.md）を生成するスキル。

## このスキルが生成するもの

ユーザーのプロダクト情報をヒアリングした上で、以下を含むDESIGN.mdを生成する。

1. デザイン哲学（3原則）
2. カラーシステム（ベース・セマンティック・アクセント + 運用ルール）
3. タイポグラフィ（フォント選定・タイプスケール・運用ルール）
4. スペーシングとレイアウト（グリッド・ブレークポイント）
5. コンポーネントスタイル（カード・ボタン・フォーム・ナビゲーション等）
6. アイコンとビジュアル
7. アニメーションとトランジション
8. やらないことリスト
9. デザインリファレンス
10. Tailwind CSS設定（実装可能なデザイントークン）

---

## 進め方

### Step 1: プロダクト情報のヒアリング

以下を確認する。不明な場合はユーザーに質問する。

- **プロダクトの概要**: 何をするプロダクトか
- **ターゲットユーザー**: 誰が使うか（開発者 / 事業担当者 / 一般消費者 / etc.）
- **利用シーン**: どんな状況で使うか（集中して分析する / 移動中にさっと確認する / etc.）
- **主要な画面構成**: ダッシュボード / エディタ / チャット / マップ / etc.
- **既存のブランドカラー**: あれば活用する。なければ提案する
- **技術スタック**: Tailwind CSS / shadcn/ui / Chakra UI / etc.（デザイントークンの出力形式に影響）
- **デザインの好み・参考プロダクト**: あれば聞く
- **明確に避けたいもの**: AIっぽいデザイン / ダークモード / etc.

### Step 2: デザイン方針の決定

ヒアリング結果を踏まえ、3つのデザイン原則を定義する。以下をデフォルトの土台とし、プロダクトの性質に応じて調整する。

---

## デザインの土台：Calm Design + Strategic Minimalism

### デフォルトの3原則

**原則1：すべての要素が存在理由を持つ（Strategic Minimalism）**

画面上のすべての要素は、ユーザーを目標に近づけるために存在しなければならない。装飾的なグラデーション、不要なアイコン、飾りのイラストは排除する。余白（ホワイトスペース）は注意を集中させるための機能的な要素として扱う。

Vercelのデザインシステムに倣い、余白は「詰める」より「広げる」方向で判断する。余白の広さが信頼感と品質の印象を生む。

**原則2：AIは透明であること（Invisible AI）**

AIを搭載するプロダクトであっても、AIの存在を誇示しない。「AI搭載」「AI分析中」のようなバッジやラベルは使わない。AIの提案はインターフェースに自然に溶け込み、ツール自体が賢いかのように振る舞う。チャットバブルやロボットアイコンは避け、AIサジェスションは控えめな帯やインラインテキストで表現する。

**原則3：情報の階層はタイポグラフィで導く（Typography-driven Hierarchy）**

色やアイコンではなく、文字の大きさ・太さ・濃淡で情報の優先順位を伝える。ユーザーの視線を自然にガイドし、最も重要な情報が最初に目に入るようにする。

### 原則の調整が必要なケース

| プロダクトの性質 | 調整方針 |
|---|---|
| エンタメ・ゲーム系 | 原則1を緩和。装飾やアニメーションを許容 |
| 開発者ツール | ダークモードをデフォルトに。Monoフォントの比重を上げる |
| BtoC / カジュアル | イラストや絵文字の限定的な使用を許容 |
| AI機能が主要な売り | 原則2を緩和。ただし「バッジ」ではなく「体験に溶け込んだAI」を目指す |

## カラーシステムの設計

### 基本構造

3層のカラーシステムを採用する。

**① ベースカラー（画面の90%以上を占める）**

| トークン名 | 用途 | ライトモードのデフォルト |
|---|---|---|
| `bg-primary` | メイン背景 | `#FFFFFF` |
| `bg-secondary` | カード・セクション背景 | `#FAFAFA` |
| `bg-tertiary` | ホバー・選択状態 | `#F5F5F5` |
| `border-default` | 区切り線・カード境界 | `#E5E5E5` |
| `border-subtle` | 控えめな区切り | `#F0F0F0` |
| `text-primary` | 本文・見出し | `#171717` |
| `text-secondary` | 補足テキスト | `#737373` |
| `text-tertiary` | プレースホルダー・無効 | `#A3A3A3` |

プロダクトのトーンに応じてグレーの色温度を調整する。Linearのようにやや暖かいグレー（黄味寄り）にするか、Vercelのように純粋なニュートラルグレーにするか。

**② セマンティックカラー（ステータス・フィードバック用）**

| トークン名 | 用途 | デフォルト値 |
|---|---|---|
| `status-success` | 成功・完了 | `#16A34A`（緑系） |
| `status-warning` | 警告・注意 | `#F59E0B`（アンバー系） |
| `status-error` | エラー・危険 | `#DC2626`（赤系） |
| `status-info` | 情報・進行中 | `#2563EB`（ブルー系） |
| `status-neutral` | 無効・未着手 | `#A3A3A3`（グレー） |

プロダクト固有のステータスがある場合は追加する（例：棄却、保留、レビュー中 等）。

**③ アクセントカラー（ブランド＋CTA用）**

1色を選定し、3つのバリエーション（通常・ホバー・薄い背景）を定義する。

| トークン名 | 用途 |
|---|---|
| `accent-primary` | CTAボタン、アクティブ状態 |
| `accent-primary-hover` | ホバー状態 |
| `accent-subtle` | 薄い背景（選択中の要素等） |

### カラー運用ルール（必ず適用する）

- ベースカラーが画面の90%以上を占めること
- セマンティックカラーはステータス表示のみに使用。装飾には使わない
- アクセントカラーはユーザーの次のアクションを示す箇所にのみ使用
- **色だけで情報を伝えない。必ずテキストラベル or アイコンを併用する**（WCAG 2.2準拠、色覚多様性への対応）
- テキストとのコントラスト比は最低4.5:1（WCAG AA）を確保
- 赤と緑を対で使う場合は、色覚多様性に配慮しブルーとオレンジの代替ペアも検討する

## タイポグラフィの設計

### フォント選定の方針

| 用途 | 推奨フォント | 選定基準 |
|---|---|---|
| UI全般（欧文） | **Inter** | 可読性、Tailwind互換性、実績（Linear, Vercel） |
| UI全般（日本語） | **Noto Sans JP** | Interとの相性、ウェイトの豊富さ、無料 |
| 数値・コード | **Geist Mono** or **JetBrains Mono** | tabular-numsで数字幅が揃う |

ブランドに固有のフォントがある場合はそちらを優先するが、可読性とウェイトのバリエーションが十分か確認する。

### タイプスケール（デフォルト）

| レベル | サイズ | ウェイト | 用途 |
|---|---|---|---|
| Display | 30px / 1.875rem | Bold (700) | 画面タイトル |
| Heading 1 | 24px / 1.5rem | Semibold (600) | セクション見出し |
| Heading 2 | 20px / 1.25rem | Semibold (600) | カードタイトル |
| Heading 3 | 16px / 1rem | Semibold (600) | サブセクション |
| Body | 14px / 0.875rem | Regular (400) | 本文 |
| Caption | 12px / 0.75rem | Regular (400) | 補足情報 |
| Mono | 14px / 0.875rem | Regular (400) | 数値表示 |

### タイポグラフィ運用ルール

- 見出しと本文の間には最低16pxのマージン
- 1行の文字数：日本語35〜45文字、英語60〜75文字
- 太字（Bold）は見出しのみ。本文中の強調はSemibold
- 数値には`font-variant-numeric: tabular-nums`を適用

## スペーシングとレイアウト

### 8pxグリッド

| トークン | 値 | 用途 |
|---|---|---|
| `space-1` | 4px | アイコンとラベルの間 |
| `space-2` | 8px | テキスト間の最小マージン |
| `space-3` | 12px | コンパクトな要素間 |
| `space-4` | 16px | カード内パディング |
| `space-5` | 24px | セクション間 |
| `space-6` | 32px | 大きなセクション区切り |
| `space-8` | 48px | ページレベルの区切り |
| `space-10` | 64px | メインコンテンツの上下余白 |

### レスポンシブブレークポイント

| ブレークポイント | 幅 | レイアウト |
|---|---|---|
| Desktop | ≥1024px | サイドバー + メインの2カラム |
| Tablet | 768px〜1023px | サイドバー折りたたみ、1カラム |
| Mobile | <768px | 完全1カラム |

## コンポーネントスタイルの設計

### 共通ルール

- **影（box-shadow）は使わない**。ボーダーで領域を区切る。影は視覚的な重みを生み、Calm Designに反する
- **角丸は8px以下**。丸すぎるとカジュアルになりすぎる
- **ホバー時の変化は控えめに**。背景色の微妙な変化 + ボーダー色の濃度アップ程度

### カード

```
背景: bg-secondary
ボーダー: 1px solid border-default
角丸: 8px
影: なし
ホバー: bg-tertiary + border色をやや濃く
ステータス表示がある場合: 左端に4pxの縦ラインでステータスカラー
```

### ボタン

| タイプ | スタイル | 用途 |
|---|---|---|
| Primary | accent-primary背景 + 白テキスト。角丸6px | 主要アクション |
| Secondary | 白背景 + border-default + text-primary | 補助アクション |
| Ghost | 背景なし + text-secondary。ホバー時bg-tertiary | 控えめなアクション |
| Danger | status-error背景 + 白テキスト | 破壊的アクション（確認ダイアログ必須） |

ボタン高さ: 36px（標準）、44px（モバイルタッチターゲット）

### AIサジェスション（AI搭載プロダクトの場合）

```
背景: bg-secondary
左端に2pxの accent-primary ライン
テキスト: text-secondary（控えめ）
「AIが分析しました」のようなラベルは付けない
「💡」アイコン + 提案内容をそのまま表示
```

## アイコン

### 推奨ライブラリ

**Lucide Icons**
- Linearストロークスタイル（1.5pxストローク幅）
- React対応（`lucide-react`）
- オープンソース

### 運用ルール

- アイコンは必ずテキストラベルとセットで使用
- サイズ: 20px（標準）、24px（ナビゲーション）
- カラー: text-secondary。アクティブ時text-primary
- イラスト・装飾画像は原則使わない。Empty Stateはテキスト + アクションボタンで対応

## アニメーション

### 基本方針

アニメーションは「存在に気づかないレベル」で使用する。状態変化を自然に伝える目的のみ。

| 対象 | duration | easing |
|---|---|---|
| ホバー | 150ms | ease-out |
| 展開/折りたたみ | 200ms | ease-in-out |
| ページ遷移 | 0ms（即時） | — |

### 禁止事項

- バウンス、スプリング等のplayfulなイージング
- フェードイン/スライドインでの要素出現（即時表示）
- ローディング時のパルスアニメーション（スケルトンスクリーンを優先）

---

## やらないことリスト（デフォルト）

プロダクトの性質に応じて調整するが、以下をデフォルトの禁止事項とする。

| やらないこと | 理由 |
|---|---|
| グラデーション背景 | Calm Designに反する |
| AI/ロボットのアイコン・イラスト | AIっぽさの排除 |
| 「AIが分析中」バッジ・ラベル | AIは透明であるべき |
| ダークモードのネオンカラー | 開発者ツール的な印象 |
| 16px以上の角丸 | カジュアルすぎる |
| カード上の影（box-shadow） | 視覚的な重さを生む |
| 「🚀 さあ始めよう！」系コピー | テックスタートアップ的な軽さ |
| 複雑なチャート・グラフ（必要時以外） | シンプルな数値表示で十分な場合が多い |

## デザインリファレンス

デザイン判断に迷ったとき、以下のプロダクトを参照する。

| プロダクト | 参照ポイント |
|---|---|
| **Linear** | Calm Design全体。余白。タイポグラフィ中心の情報階層。暖かみのあるグレー |
| **Vercel** | 極限まで削ぎ落としたカラーパレット。余白の広さが生む品質感。ボーダーの控えめさ |
| **Notion** | AIの透明な統合。機能の豊富さを感じさせないシンプルUI |
| **Stripe（ダッシュボード）** | アクセシブルなカラーシステム。数値表示のタイポグラフィ |
| **Figma** | コマンドパレット（Cmd+K）。複雑な機能を隠しつつアクセス可能にする設計 |

## 出力形式

### DESIGN.mdの構成

生成するDESIGN.mdは以下の構成とする。

1. デザイン哲学（プロダクト固有の3原則）
2. カラーシステム（ベース・セマンティック・アクセント + 具体的なHEX値 + 運用ルール）
3. タイポグラフィ（フォント・タイプスケール・運用ルール）
4. スペーシングとレイアウト（グリッド・ブレークポイント）
5. コンポーネントスタイル（プロダクトで使う主要コンポーネントのスタイル定義）
6. アイコンとビジュアル
7. アニメーションとトランジション
8. やらないことリスト
9. デザインリファレンス
10. 実装用デザイントークン（Tailwind CSS config等、技術スタックに合わせた出力）

### 技術スタック別の出力

| スタック | 出力形式 |
|---|---|
| Tailwind CSS | `tailwind.config.ts`のtheme.extend設定 |
| CSS Variables | `:root`に定義するCSS Custom Properties |
| shadcn/ui | `globals.css`のCSS変数 + `tailwind.config.ts` |
