# リアルタイム検知（Webhook）移行記録

**実施日**: 2025年12月26日

## 概要

これまでCloud Run + Cloud Schedulerによる「1分ごとのポーリング」で行っていたGoogle Driveのファイル検知を、**Google Drive Push Notifications (Webhook)** によるリアルタイム検知に移行しました。

これにより、以下のメリットが得られます：
1.  **コスト削減**: 無駄なポーリングリクエスト（特に深夜帯）の廃止。
2.  **即時性**: ファイルアップロードから検知までのタイムラグを短縮。

## アーキテクチャ変更点

### Before (ポーリング)
- Cloud Schedulerが1分ごとに `/api/grading/check` を叩く
- Drive APIでファイルリストを取得し、新着ファイルをチェック

### After (Webhook)
- **検知**: Google Driveがファイル追加時に `/api/grading/webhook` に通知
- **自動更新**: Google DriveのWatch有効期限（約24時間）を維持するため、Cloud Schedulerが12時間ごとに `/api/drive/watch/renew` を叩く

## 新規・変更コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| API | `/api/drive/watch/setup` | 初回Watch登録用（手動実行） |
| API | `/api/drive/watch/renew` | Watch自動更新用（Schedulerから実行） |
| API | `/api/grading/webhook` | 通知受信用（既存エンドポイントを流用・改修） |
| Lib | `src/lib/drive-watch-state.ts` | Watch状態（ID, 有効期限）をUpstash Redisに保存 |
| Cloud | Scheduler `sullivan-drive-watch-renew` | 12時間ごとに更新APIを実行 |

## 必要な環境変数

- `APP_URL` (Webhook受信に必要な公開URL)
- `DRIVE_FOLDER_ID`
- `UPSTASH_REDIS_REST_URL` / `UPSTASH_REDIS_REST_TOKEN`
- `DRIVE_WEBHOOK_CHANNEL_ID` (任意: Webhookの検証に使用)
- `INTERNAL_API_SECRET` (任意: `/api/grading/check` を使う場合)

## 運用上の注意点

### ドメイン認証
Webhookを受信するためには、Google Search ConsoleにてCloud Runのドメイン所有権が確認されている必要があります。
検証用ファイル: `public/googlef2680590528c261b.html`

### Watchの有効期限
Watchは最大でも約24時間で切れます。Schedulerによる自動更新が失敗し続けると検知が止まるため、ログ（`/api/drive/watch/renew`）の監視が推奨されます。
万が一止まった場合は、`/api/drive/watch/setup` を叩くことで再登録可能です。
