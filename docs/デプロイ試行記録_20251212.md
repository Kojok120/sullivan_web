# Deployment Walkthrough (Cloud Run + Supabase + Upstash)

本ドキュメントは、現行実装（Drive Webhook + QStash）に合わせて内容を整理したデプロイ手順です。

## 1. 事前準備 (Action Items)

### A. Supabase (Database + Auth)
1. Supabaseダッシュボードの **Connect** -> **URI** を開く。
2. **Transaction Mode (Port 6543)** を `DATABASE_URL` に使用（末尾に `?pgbouncer=true`）。
3. **Session Mode (Port 5432)** を `DIRECT_URL` に使用。
4. **Project URL / Anon Key / Service Role Key** を取得。

### B. Upstash (QStash & Redis)
1. Upstash Console で **Redis Database** を作成。
    * `UPSTASH_REDIS_REST_URL` と `UPSTASH_REDIS_REST_TOKEN` を取得。
2. **QStash** で以下を取得。
    * `QSTASH_TOKEN`
    * `QSTASH_CURRENT_SIGNING_KEY`
    * `QSTASH_NEXT_SIGNING_KEY`

### C. Google Cloud Project & Secret Manager
1. Google Drive API を有効化。
2. サービスアカウントキー (`service-account.json`) を作成。
3. Secret Manager にシークレットを登録し、Cloud Runのサービスアカウントへアクセス権を付与。
4. Cloud Runに `/secrets` へマウントし、`GOOGLE_APPLICATION_CREDENTIALS=/secrets/google-creds` を設定。

### D. APP_URL の決定
Cloud RunのURLが必要です。初回は仮の `APP_URL` を入れてデプロイし、
URL確定後に `APP_URL` を更新して再デプロイしてください。

## 2. 必要な環境変数

以下は本番環境で必要な環境変数の一覧です。

- `DATABASE_URL`
- `DIRECT_URL`
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `GEMINI_API_KEY`
- `DRIVE_FOLDER_ID`
- `APP_URL`
- `QSTASH_TOKEN`
- `QSTASH_CURRENT_SIGNING_KEY`
- `QSTASH_NEXT_SIGNING_KEY`
- `UPSTASH_REDIS_REST_URL`
- `UPSTASH_REDIS_REST_TOKEN`
- `GOOGLE_APPLICATION_CREDENTIALS`
- `INTERNAL_API_SECRET` (任意: `/api/grading/check` 用)
- `DRIVE_WEBHOOK_CHANNEL_ID` (任意: Webhook検証用)

## 3. デプロイ実行コマンド (Secrets利用版)

```bash
# 1. Google Cloud 認証
gcloud auth login
gcloud config set project [YOUR_PROJECT_ID]

# 2. Cloud Run へデプロイ
gcloud run deploy sullivan-app \
  --source . \
  --platform managed \
  --region asia-northeast1 \
  --allow-unauthenticated \
  --set-env-vars "NODE_ENV=production" \
  --set-env-vars "DATABASE_URL=postgres://..." \
  --set-env-vars "DIRECT_URL=postgres://..." \
  --set-env-vars "NEXT_PUBLIC_SUPABASE_URL=https://..." \
  --set-env-vars "NEXT_PUBLIC_SUPABASE_ANON_KEY=..." \
  --set-env-vars "SUPABASE_SERVICE_ROLE_KEY=..." \
  --set-env-vars "GEMINI_API_KEY=..." \
  --set-env-vars "DRIVE_FOLDER_ID=..." \
  --set-env-vars "QSTASH_TOKEN=..." \
  --set-env-vars "QSTASH_CURRENT_SIGNING_KEY=..." \
  --set-env-vars "QSTASH_NEXT_SIGNING_KEY=..." \
  --set-env-vars "UPSTASH_REDIS_REST_URL=..." \
  --set-env-vars "UPSTASH_REDIS_REST_TOKEN=..." \
  --set-env-vars "APP_URL=https://[YOUR-CLOUD-RUN-URL]" \
  --set-env-vars "GOOGLE_APPLICATION_CREDENTIALS=/secrets/google-creds" \
  --update-secrets=/secrets/google-creds=sullivan-service-account:latest
```

※ `INTERNAL_API_SECRET` / `DRIVE_WEBHOOK_CHANNEL_ID` を使う場合は `--set-env-vars` に追加してください。

## 4. データベースのセットアップ (初回のみ)

```bash
npx prisma migrate deploy
npx prisma db seed
```

### トラブルシューティング: DBマイグレーションが止まる場合
Supabaseの接続プールの影響で `migrate deploy` が止まる場合は、
`DIRECT_URL` を使って実行してください。

```bash
export $(grep DIRECT_URL .env | xargs) && DATABASE_URL=$DIRECT_URL npx prisma migrate deploy
export $(grep DIRECT_URL .env | xargs) && DATABASE_URL=$DIRECT_URL npx prisma db seed
```

## 5. Drive Webhook セットアップ

デプロイ後に以下を実行し、Drive Watchを登録します。

```bash
curl -X POST https://[YOUR-CLOUD-RUN-URL]/api/drive/watch/setup
```

Watchは約24時間で失効するため、Cloud Scheduler等で
`POST /api/drive/watch/renew` を12時間おきに実行してください。

## 7. トラブルシューティング (現行実装)

- **Driveファイルが `[ERROR]` になる**
  - QRのIDがDBに存在しない場合に発生します。本番環境で発行したQRを使用してください。
- **Webhookが発火しない**
  - `APP_URL` が正しいか、Search Consoleのドメイン認証が済んでいるか確認してください。
  - `UPSTASH_REDIS_REST_*` 未設定の場合、Watch状態の保存に失敗します。
- **QStashを使わず同期処理になってしまう**
  - `QSTASH_TOKEN` が設定されていないと同期処理にフォールバックします。
- **Supabase Authユーザーがいない**
  - `SUPABASE_SERVICE_ROLE_KEY` を設定した上で `prisma db seed` を実行してください。
